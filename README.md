# 🥚 TamaBox (타마박스)

> **하루를 키우는 타임박스** — 타임박스로 하루를 계획하고, 카테고리(운동·언어공부·주식공부 등)별로 계획을 얼마나 지켰는지 **일·주·월** 단위로 추적해요. 다마고찌 스타일 친구를 키우며(밥주기·진화·도감) 꾸준함을 쌓는 귀여운 앱.

## ✨ 기능

- **타임박스 플래너** — 하루를 30분(조절 가능) 칸으로 나누고, 카테고리 색으로 칸을 **드래그해서 칠하면** 계획 완성
- **체크 모드** — 실천한 칸을 눌러 ✅ 체크. "계획을 지켰는지" 바로 기록
- **카테고리(스킬) 관리** — 이름·이모지·색·하루 목표시간을 자유롭게. 기본: 주식공부 📈 / 운동 💪 / 언어공부 🗣️ / 독서 📖 / 리서치 🔍 / 휴식 ☕
- **성과 트래킹** — 달성률 추이, 카테고리별 계획 vs 실천 시간, 출석 히트맵(일/주/월 전환)
- **게임화** — 칸 완료 시 XP·레벨업, 🔥연속 달성 스트릭, 업적 배지, 효과음·색종이 애니메이션
- **클라우드 동기화** — Supabase 무료 티어로 기기 간 동기화 (선택)
- **백업** — JSON 내보내기/가져오기
- **무료 · Mac/Windows** — 빌드 없이 브라우저만 있으면 동작

## 🚀 바로 쓰기 (가장 간단)

`index.html`을 더블클릭하면 끝. 데이터는 그 브라우저에 자동 저장됩니다(오프라인).
> ※ 더블클릭 시 일부 환경에서 웹폰트/효과음만 제한될 수 있어요. 가장 안정적으로 쓰려면 아래 "로컬 서버" 또는 "무료 배포"를 권장합니다.

### 로컬 서버로 열기 (권장, 무료)
```bash
cd timebox-quest
python3 -m http.server 8080      # 또는: npx serve .
# 브라우저에서 http://localhost:8080 접속
```

## ☁️ 클라우드 동기화 설정 (선택, 무료)

여러 기기(맥·윈도우·폰)에서 같은 데이터를 쓰고 싶다면:

1. **Supabase 가입** → https://supabase.com (무료) → 새 프로젝트 생성
2. 프로젝트의 **Settings → API** 에서 `Project URL`과 `anon public key` 복사
3. **SQL Editor** 열고 이 폴더의 [`supabase.sql`](./supabase.sql) 내용을 붙여넣어 **RUN**
4. (즉시 로그인하려면) **Authentication → Sign In / Providers → Email** 에서
   `Confirm email`(이메일 확인)을 끄면 가입 후 바로 로그인됩니다. *(보안을 원하면 켜둔 채 메일 확인)*
5. 앱의 **⚙️ 설정 → ☁️ 클라우드 동기화** 에 URL·anon key 입력 후 **설정 저장**
6. 이메일·비밀번호로 **회원가입 → 로그인**. 이후 변경사항이 자동으로 동기화됩니다.

> 동기화 방식: 사용자별 JSON 문서 1개 + `updated_at` 기준 **최신 우선(last-write-wins)**.
> HUD 우측 상단의 점 색으로 상태를 확인하세요. 🟢동기화됨 / 🟡동기화중 / 🔴오류 / ⚪오프라인

### 🌐 어디서나 접속하려면 (무료 배포)
이 폴더를 그대로 무료 정적 호스팅에 올리면 URL로 접속할 수 있습니다(맥/윈도우/폰 공용):
- **Cloudflare Pages / Netlify / Vercel / GitHub Pages** 중 아무거나 — 빌드 설정 없이 폴더만 업로드
- 배포 후 그 URL에서 위 클라우드 동기화로 로그인하면 모든 기기가 같은 데이터를 공유

## 🕹️ 사용법

| 모드 | 동작 |
|------|------|
| ✏️ **계획** | 위에서 카테고리(색)를 고르고, 시간 칸을 **클릭/드래그**해 칠합니다. 🧽지우개로 제거 |
| ✅ **체크** | 실천 완료한 칸을 눌러 체크/해제 (XP 획득!) |
| **⋯** (칸 우상단) | 칸의 카테고리·메모·완료 여부를 자세히 편집 |

- 상단 화살표로 날짜 이동, **오늘로** 버튼으로 복귀
- **📊 성과** 탭에서 일/주/월 전환하며 추이·카테고리·히트맵·업적 확인
- **⚙️ 설정** 에서 카테고리, 하루 시간 범위, 칸 단위(15/30/60분), "성공한 날" 기준 달성률, 효과음, 동기화, 백업 관리

## 🗂️ 구조

```
timebox-quest/
├── index.html            # 화면 구조 (탭/플래너/성과/도감/설정/모달)
├── css/theme.css         # 모던 큐트 테마 (원색 캔디·둥근 카드·테마 변수 --app-bg)
├── js/
│   ├── state.js          # 데이터 모델 · 저장 · 마이그레이션 · 날짜 유틸 · 통계 derive
│   ├── charts.js         # 차트(막대/카테고리/히트맵)
│   ├── gamify.js         # XP/레벨/업적/토스트/효과음/색종이
│   ├── character.js      # 캐릭터 육성(고양이/강아지·진화·밥주기·도감·테마·여러마리)
│   ├── planner.js        # 타임박스 그리드(터치 칠하기/체크/복사·템플릿/실제기록)
│   ├── analytics.js      # 성과 탭(일/주/월) + 인사이트 + CSV
│   ├── sync.js           # Supabase 동기화(충돌 방지 + 포커스 시 동기화)
│   └── app.js            # 부트스트랩 · 탭 · HUD · 설정 · 모달 · 되돌리기(Undo)
├── manifest.webmanifest  # PWA(설치형) 매니페스트
├── sw.js                 # 서비스워커(오프라인 캐싱)
├── icon.svg              # 앱 아이콘
├── tests/logic.test.js   # 회귀 테스트 (node tests/logic.test.js)
├── supabase.sql          # 동기화용 테이블 + RLS
└── README.md
```

### 테스트
```bash
node tests/logic.test.js   # 핵심 로직 회귀 테스트 (DOM 스텁)
```

### 📲 앱으로 설치 (PWA)
호스팅한 URL(또는 로컬 서버)에서 열면 브라우저 메뉴의 **"홈 화면에 추가 / 앱 설치"** 로 설치할 수 있어요.
서비스워커가 화면·폰트를 캐싱해 **오프라인에서도** 열립니다. (file:// 더블클릭에선 설치/오프라인 비활성 — 로컬 서버나 배포 URL 필요)

데이터는 브라우저 `localStorage` 의 `timeboxQuest.v1` 키에 저장됩니다.

## 💡 전업투자자용 활용 팁

- 장 시작 전 **리서치/주식공부** 칸을 미리 깔아두고, 장중엔 **체크 모드**로 실천 기록
- 카테고리별 **하루 목표시간**을 설정해 "주식공부 2시간" 같은 루틴을 게임 목표로
- 주말에 **📊 성과 → 월** 로 카테고리 균형 점검 (한쪽에 치우치지 않았는지)
- 🔥스트릭을 끊지 않는 것을 1차 목표로 — 작은 꾸준함이 누적됩니다
